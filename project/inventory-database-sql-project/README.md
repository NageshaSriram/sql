# Inventory Management Analysis Project (MySQL)

## Objective

Perform SQL-based data analysis on an inventory management dataset to extract insights, demonstrate SQL proficiency, and prepare a report and presentation for MTech evaluation.

## Dataset Description

This dataset contains product, customer, employee, warehouse, region, and order information. It supports business analytics, inventory optimization, and performance tracking.

## Dataset Features

- Product.csv: Includes product names, categories, standard costs, and list prices.

- Orders.csv: Contains order details such as quantities, unit prices, and order statuses.

- Customer.csv: Customer information, including names, contact details, and credit limits.

- Warehouse.csv: Data on warehouse locations and inventory distribution.

- Employee.csv: Employee records linked to warehouse operations.

- Region.csv: Geographic data, including country, state, city, and postal codes.

- OrderDetails.csv: Transaction-level data connecting products and orders, detailing sales volume and revenue.

This dataset is ideal for SQL queries, inventory performance analysis, supply chain management insights, and business analytics.

## Steps

1. Import CSVs into MySQL.
2. Explore schema and relationships.
3. Perform descriptive, aggregate, and advanced SQL analyses.
4. Use subqueries, CTEs, and window functions.
5. Derive insights and prepare presentation slides.

## Important SQL scripting files

- `schema_and_load.sql` - Schema Creation & Data Import

- `analysis` - Business Analysis including analytics, inventory optimization, and performance tracking.
